# R语言教程编写器

这个skill用于编写R语言教程的R Markdown文档，遵循以下规范：

## 文档结构要求

### YAML头部
```yaml
---
title: "教程标题"
author: "医学研究生救星计划"
output: html_document
---
```

### Setup代码块
```r
```{r setup, include=FALSE}
knitr::opts_chunk$set(echo = TRUE)
```
```

## 代码编写规范

### 1. 代码注释要求
- **每个代码块必须有详细注释**
- 注释使用简单易懂的语言，适合R语言初学者
- 关键函数需要说明参数含义
- 复杂逻辑需要分步注释
- 注释使用中文

### 2. 注释风格示例

```r
# 创建数值向量x，包含1到5
x <- c(1, 2, 3, 4, 5)

# 计算向量x的平均值
mean_x <- mean(x)

# 输出结果
mean_x
```

### 3. 代码块格式
- 不要在代码块中手动写输出结果（如 `##>` 格式）
- R在knit时会自动执行代码并显示输出
- 使用 `eval=FALSE` 标记不执行的代码块

### 4. 表格格式
- 表格标题后需要空一行
- 使用标准Markdown表格格式：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

### 5. 章节结构
- 使用 `#` 表示章
- 使用 `##` 表示节
- 使用 `###` 表示小节
- 章节之间用 `---` 分隔

## 内容编写原则

### 语言风格
- 使用简单易懂且生动的语言
- 避免使用初学者不理解的专业术语
- 如需使用专业术语，必须在相应位置进行解释

### 教学方式
- 从简单到复杂，循序渐进
- 每个知识点配有可执行的代码示例
- 提供实际应用场景说明
- 使用表格总结关键知识点

### 代码示例
- 所有代码必须可执行（除非标记 `eval=FALSE`）
- 示例要贴近实际应用
- 避免过于抽象的示例

## 常见问题处理

### 逻辑运算符
- `&&` 和 `||` 只能用于单个逻辑值（if条件判断）
- `&` 和 `|` 用于向量化运算
- 新版本R中 `&&` 不能直接用于向量

### 文件路径
- Windows路径使用正斜杠 `/` 或双反斜杠 `\\`
- 不能使用单反斜杠 `\`

### 表格渲染
- 表格前必须有空行
- 分隔符使用 `|---|` 格式，不是中文破折号

## 输出格式

生成的Rmd文件应包含：
1. 完整的YAML头部
2. Setup代码块
3. 章节标题和内容
4. 带详细注释的代码块
5. 总结表格
6. 章节分隔线

## 使用示例

当用户要求编写R语言教程内容时，按照以上规范：
1. 创建或更新指定的 .Rmd 文件
2. 添加详细注释
3. 确保代码可执行
4. 使用简单易懂的语言解释概念
